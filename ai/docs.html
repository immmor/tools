🔑 认证方式

immmorAI 使用 Bearer Token 进行 API 认证。

获取 Token

在管理后台的"钥匙包"页面创建新的 Token。每个 Token 可以设置不同的额度和权限。

# 在请求头中添加 Authorization
Authorization: Bearer your_token_here

安全提示

  • 不要在客户端代码中暴露 Token
  • 使用环境变量存储敏感信息
  • 定期轮换 Token 以保证安全
  • 为不同环境创建不同的 Token

💻 代码示例

流式响应示例 (Python)

import requests
import json

url = "https://api.immmor.com/v1/chat/completions"
headers = {
    "Authorization": "Bearer YOUR_TOKEN",
    "Content-Type": "application/json"
}

data = {
    "model": "gpt-3.5-turbo",
    "messages": [{"role": "user", "content": "讲个故事"}],
    "stream": True
}

response = requests.post(url, headers=headers, json=data, stream=True)

for line in response.iter_lines():
    if line:
        line = line.decode('utf-8')
        if line.startswith('data: '):
            data = line[6:]
            if data != '[DONE]':
                content = json.loads(data)['choices'][0]['delta'].get('content', '')
                print(content, end='', flush=True)

完整对话示例 (Node.js)

const axios = require('axios');

async function chat() {
    const messages = [
        { role: "system", content: "你是一个有帮助的助手" },
        { role: "user", content: "你好！" }
    ];

    try {
        const response = await axios.post(
            'https://api.immmor.com/v1/chat/completions',
            {
                model: 'gpt-3.5-turbo',
                messages: messages
            },
            {
                headers: {
                    'Authorization': 'Bearer YOUR_TOKEN',
                    'Content-Type': 'application/json'
                }
            }
        );

        console.log(response.data.choices[0].message.content);
    } catch (error) {
        console.error('Error:', error.message);
    }
}

chat();

❓ 常见问题

如何充值？

在个人中心点击"充点小钱钱"按钮，即可跳转到充值页面。支持支付宝、微信等多种支付方式。

Token 额度不够怎么办？

可以在"钥匙包"页面为现有 Token 增加额度，或者创建新的 Token。每个 Token 的额度是独立的。

遇到错误怎么办？

首先检查 API 返回的错误信息。常见问题包括：Token 无效、额度不足、请求格式错误等。如果问题持续，请联系技术支持。

如何查看使用记录？

在"钥匙包"页面可以看到每个 Token 的详细使用记录，包括调用次数、消耗金额等信息。

API 有调用限制吗？

为了保证服务质量，API 有速率限制。具体限制取决于你的账户等级。如需更高限额，请联系客服。
